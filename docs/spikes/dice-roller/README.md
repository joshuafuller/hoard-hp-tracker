# Spike: 3D dice roller (proven)

A throwaway-but-saved spike that proves physics-based 3D dice (D&D-Beyond-style)
for Hoard, ahead of the real integration (issue #73). **Not a text roller**, and
we never model dice ourselves.

## Run it
```bash
cd docs/spikes/dice-roller
./setup.sh                 # installs the libs, copies dice-box dist, bundles the parser
python3 -m http.server 8900   # → http://localhost:8900/
```
`dist/`, `parser.js`, and `node_modules/` are generated by `setup.sh` and are
git-ignored — only `index.html` + `setup.sh` are committed.

## Engine decision
- **[@3d-dice/dice-box](https://github.com/3d-dice/dice-box) v1.1.4** — rendering + physics
  (BabylonJS + Ammo, web-worker/offscreen-canvas). Ships its own dice meshes + themes;
  we gold-tint via `themeColor: "#e8b45a"`.
- **[@3d-dice/dice-parser-interface](https://github.com/3d-dice/dice-parser-interface) v0.2.1** —
  full Roll20 notation (dice-box alone only does `NdX±mod`). Flow:
  `parseNotation()` → `box.roll()` → in `onRollComplete`: `handleRerolls()` (loop while any) → `parseFinalResults()`.

## Decided interface (see #73)
Full-screen **"table throw"**: a d20 gold token in the chrome (later the radial
purse) dims the app and turns the whole screen into a transparent dice tray;
dice tumble over the dimmed card. Choose via die chips or a notation field;
**total + per-die result + roll history**; tap the tray to clear, ✕ to close.
The app's existing **death-save d20** and **short-rest Hit Dice** roll through
this same tray (one shared mechanic).

## Notation supported (proven)
| Want | Notation |
|---|---|
| Advantage / Disadvantage | `2d20kh1` / `2d20kl1` |
| Stat (4d6 drop lowest) | `4d6kh3` (or `4d6dl1`) |
| Exploding / compounding / penetrating | `3d6!` / `!!` / `!p` |
| Reroll / reroll-once | `4d6r1` / `ro1` |
| Success counting | `6d6>4` |
| Fudge/Fate | `4dF` |
| Math | `1d20+5`, `2d6+1d4+3` |

## Recorded result model (per roll)
```js
{ notation: "4d6kh3", total: 16, result: [4,6,6], dice: [
    {sides:6,value:4,dropped:false}, {sides:6,value:6,dropped:false},
    {sides:6,value:6,dropped:false}, {sides:6,value:3,dropped:true} ] }
```
This is the shape the app should persist (powers a roll log + the death-save / Hit-Dice records).

## Lessons for the real build (carry into #73)
1. **Serve dice-box + assets same-origin.** CDN worker/WASM imports hit CORS traps. For the app: copy `dist/assets` (themes + `ammo.wasm`) and `world.*.js` into `public/` and **precache via Workbox** so rolling works offline (#45).
2. **Bundle `dice-parser-interface` locally** (esbuild). A CDN/esm.sh import threw at module top-level and *silently killed the whole script* (handlers never attached → "click does nothing, no errors").
3. **Overlays must be inert when closed** (`display:none`, not just `opacity:0`) or an invisible full-screen layer eats taps on the app underneath.
4. **Lazy-load dice-box on first open** — Babylon is heavy; never on app start (keep the HP screen instant).
5. **Use `onRollComplete` + the rerolls loop**, not the simple `roll()` promise — exploding/rerolls add dice mid-roll; reconcile, then `parseFinalResults`.
